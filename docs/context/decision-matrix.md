# Матрица принятия решений

> Определяет уровень автономии агентов для каждого типа действия. Агент ОБЯЗАН проверить свои полномочия перед выполнением.

## Уровни автономии

| Уровень | Описание | Действие агента |
|---------|----------|-----------------|
| **auto** | Полная автоматизация | Агент выполняет самостоятельно, результат доступен сразу |
| **suggest+approve** | Агент предлагает, человек утверждает | Агент создаёт draft, помечает "REQUIRES APPROVAL", ждёт ревью |
| **manual only** | Только человек | Агент может предоставить информацию, но НЕ создаёт артефакт. Эскалирует |

## Матрица действий

| # | Действие | Уровень | Кто апрувит | Условие ослабления | Примечание |
|---|----------|---------|-------------|-------------------|------------|
| 1 | Генерация user story | suggest+approve | PO | После 20 успешных approve подряд → обсудить auto | Всегда ревью PO. Агент создаёт draft с тегом `[DRAFT]` |

## Правила эскалации

1. **Нет в матрице** → manual only (по умолчанию, эскалация на PO)
2. **Конфликт между constraints** → manual only, эскалация на Compliance + Tech Lead
3. **Недостаточно контекста** → агент СПРАШИВАЕТ, не додумывает
4. **Агент не уверен в уровне** → suggest+approve (безопасный default)

## Процесс ослабления

1. Ведётся лог всех approve/reject по каждому типу действия
2. При достижении порога (20 подряд без reject) — BA Lead/Tech Lead обсуждают ослабление
3. Решение фиксируется как ADR
4. Ослабление НЕ применяется к: AML, compliance, бизнес-правилам, glossary
