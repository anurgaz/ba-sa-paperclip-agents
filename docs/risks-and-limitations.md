# Риски и ограничения

> AI-native BA/SA агент для B2B карточного процессинга — известные риски и стратегии митигации.

---

## R-001: Галлюцинации в регуляторном контексте

Агент может уверенно генерировать некорректные правила PSD2, PCI DSS или AML. Ошибка в комплаенс-артефакте — это штрафы или отзыв лицензии.

**Митигация:** `constraints.md` работает как жёсткий фильтр. Всё, что касается AML/compliance, имеет уровень `manual only` в decision-matrix — решения принимает только MLRO.

## R-002: Зависимость от качества контекста

Агент настолько хорош, насколько хороша база знаний. Неполный glossary или устаревший constraints.md = системные ошибки в каждом артефакте.

**Митигация:** Аудит документации раз в спринт. Правки от ревьювера сохраняются в `/examples` как few-shot. У каждого .md файла есть владелец, ответственный за актуальность.

## R-003: Cold start

Агент не знает неписаные правила команды: naming conventions, неформальные договорённости, исторические решения без ADR. Первые недели будет много правок.

**Митигация:** Первые 2–4 недели — полный ревью всех артефактов. Каждая правка формирует `/examples`. К концу месяца агент калибруется через накопленную базу эталонов.

## R-004: Стоимость API при масштабировании

Каждый запуск загружает контекст в API. Рост базы знаний = рост стоимости. Self-healing (до 3 попыток) умножает расход.

**Митигация:** Мониторинг cost per artifact. Загрузка только релевантных файлов через `--context`. Кэширование неизменяемого контекста. Budget cap на уровне pipeline.

## R-005: Единая точка отказа — Claude API

Если API недоступен, pipeline стоит. Нет fallback — нет артефактов.

**Митигация:** Промпты модель-агностичны. Переключение на другой LLM = замена endpoint в `run-agent.sh`. Документация и валидация работают полностью независимо от API.

## R-006: Ложное чувство безопасности

Автовалидация ловит формальные ошибки (пропущенные секции, отсутствие constraint-ссылок), но не логические. Может создать иллюзию, что артефакт «проверен».

**Митигация:** Валидация = фильтр перед ревью, не замена ревью. Ни один артефакт не попадает в прод без одобрения человека. Decision-matrix явно запрещает `auto` для критичных действий.
